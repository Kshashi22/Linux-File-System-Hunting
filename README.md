# 🔍 Linux File System Hunting

This project explores the Linux file system by investigating important system files and directories such as /etc and /proc.

---

## 🧠 Objective
To understand how Linux stores configuration, system data, and real-time information using virtual and static files.

---

## 📌 Key Findings

### 📁 /etc/hosts
- Maps hostnames to IP addresses  
- Used for local DNS resolution  

### 🌐 /etc/resolv.conf
- Stores DNS server configuration  
- Helps resolve domain names  

### 🧠 /proc/cpuinfo
- Shows CPU details (model, cores, speed)  
- Generated in real-time by kernel  

### 💾 /proc/meminfo
- Displays memory usage  
- Helps detect system performance issues  

### 📊 /proc/loadavg
- Shows system load (1, 5, 15 min)  
- Indicates CPU workload  

### 🌍 /proc/net/route
- Displays routing table  
- Shows how network traffic is routed  

### 🖥️ /etc/os-release
- Contains OS identity (Ubuntu version, codename)  

### ⚙️ /etc/nsswitch.conf
- Controls how system resolves names (files, DNS, etc.)  

---

## 💡 Key Insight
Linux treats many system components as files. Even CPU, memory, and networking data can be accessed using file paths.

---

## 📊 Conclusion
Linux file system provides powerful real-time insights through virtual files.  
This exploration improved my understanding of system internals, performance monitoring, and networking behavior.  
Overall, it shows how Linux simplifies complex operations using a file-based approach.
